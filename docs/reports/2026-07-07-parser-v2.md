## 내일 클라라 실행 순서 (런타임 검증)
1. 레포의 새 `gas/Code.gs`를 PWR-CS-Engine에 덮어붙이기.
2. `cs/booking` 라벨의 실물 3통에 대해 각각 `debugPeekLatest()`(최신 1통) 또는 임시로 `pollCsInbox` 1회 실행(urlfetch 쿼터 여유 확인 후) → 로그에서 채널별 guest/bookingId/message/checkin·out 정확 추출 확인.
3. 비대상 발신자 메일이 있으면 "비대상 발신자 스킵" 로그 + `cs/inbox` 미적재 확인.
4. 3통이 `cs/drafts`에 초안 생성되는지, 익스피디아 초안은 승인·발송 시 **error("파트너센트럴에서 복붙")**로 처리되는지 확인.
5. 아고다 메시지가 이력 없이 첫 블록만 깔끔히 잡혔는지 눈으로 확인(위 ⚠️).
