# PWR CS 자동화 — 설계 확정본 v3
## 2026-07-04 (토) 갱신 / 원본: v2(7/2) + 이후 확정 사항 전부 반영
## 상태: 설계 확정 → M1 착수 대기

---

## 0. 한 줄 정의
OTA 게스트 메시지 → 클라라 말투로 답변 초안 생성 → 승인(또는 야간 자동) 발송.
클라라의 실제 답변을 계속 학습해 점점 클라라처럼 대답하는 시스템.

## 1. 포지셔닝 (왜 만드나 — 재정의됨)
- OTA 알림메일 자체가 **3~10분 지연** (클라라 실측). 고객과 소통이 끝난 뒤에야 메일이 오는 경우도 많음.
- 따라서 이 시스템의 가치는 **속도 경쟁이 아니라 커버리지**: 클라라가 자거나, 운전하거나, 전시장에 있을 때 5~10분 뒤에라도 답이 나가는 것 vs 몇 시간 무응답.
- **킬러 유스케이스 = 야간·부재 자동 응대.**
- 낮에 클라라가 익스트라넷에서 먼저 답한 건 → 앱에서 **"이미 처리" 원탭 스킵** (원문은 코퍼스 후보로 보관).

## 2. 확정 원칙 (클라라 결정)
- **완전 분리**: 별도 레포(pwr-clair/cs) + 별도 GAS(PWR-CS-Engine) + 별도 html. HK와는 버튼+뱃지 하나로만 연결.
- **학습 중심**: 정형 템플릿 선택이 아니라 클라라 답변 코퍼스를 계속 학습(모방→고도화).
- **토글 지배**: '영구 승인 카테고리' 없음. 자동발송 = 단일 토글, 클라라가 on/off.
- **자동 = 진짜 자동**: 토글 ON = **야간창(00:00–08:00) 내 전건 즉시 발송, 승인 큐 없음.** 그 외 시간은 승인. confidence 안전판은 **보류** — 무수정 승인율이 충분히 오른 뒤 재논의.
- **번역**: 게스트 언어로 직접 생성 + 한국어 대역 병기. **영어 외 모든 언어**에 번역 면책 문구 자동 첨부.

## 3. 아키텍처
```
OTA(부킹/아고다/익스피디아) 알림메일
   ↓ Gmail (라벨/필터)
   ↓ GAS "PWR-CS-Engine" 시간트리거 폴링 (1분)
   ↓ 파싱 → Firebase cs/inbox
   ↓ Claude API 초안 생성 (코퍼스 유사사례 few-shot)
   ↓ cs/drafts (+ 폰 푸시 — 채널 미확정, 텔레그램 추천)
   ↓ 승인 웹앱 (pwr-clair/cs, GitHub Pages, 모바일 우선, 기존 Firebase Auth)
   ↓ [승인 or 야간 자동] → Gmail 회신 발송 → cs/sent
```
- Firebase: 기존 프로젝트 공유, `cs/*` 네임스페이스만 사용. **app/* 쓰기 금지** (suggestions 승인 반영 제외).
- 발송 도달: OTA 알림메일의 회신 경로로 게스트 도달 확실 (클라라 확인). M1에서 가벼운 1회 확인만.

## 4. 학습 시스템 (핵심)
- **코퍼스** = ①CS-DB 스프레드시트 ②Gmail 과거 응대 스레드 마이닝 ③manual/landing 정적 사실 ④앞으로의 모든 발송분(자동 적재).
- **학습 루프**: 무수정 승인 → 정답 예시 적재 / **수정 후 발송 → (초안, 수정본) 쌍 저장** ← 말투 학습의 핵심 연료.
- 실체: 파인튜닝 아님. 유사사례 검색 + few-shot 주입. 코퍼스가 쌓일수록 고도화.
- **지표: 무수정 승인율** — 승인앱에 상시 표시. 자동발송 켤 타이밍의 객관 근거 (예: 90%+ 지속).

## 5. 업무(태스크) 탭
- 답변 속 약속 중 **기존 자동화(s1~s6)가 커버하지 않는 것**을 업무로 추출 (예: "익스피디아 이중결제 확인 후 오늘 중 회신").
- 단, "절대 금지"가 아님 — **커버되는 내용이라도 부연 설명·개별 안내가 필요하면 업무화. 원칙은 최~소화.**
- cs/tasks: 마감순 정렬, 완료 체크, 수동 추가, 마감 임박 푸시.

## 6. HK 연결 & 부가 자동화
- **ETA**: 도착시간 감지 → cs/suggestions → 승인 시 HK checkinTime 반영.
- **특별요청**: 감지 → cs/suggestions → 승인 시 HK 객실 특이사항(notes) 반영.
- **후기요청**: 대화 감성 + 응답 원활 게스트만, 체크아웃 익일, cs/reviewQueue → 토글 따라 자동/승인 발송. OTA 정책 허용 문구.
- HK 대시보드: CS 버튼 + 대기건수 뱃지만 (읽기전용).

## 7. cs/ 네임스페이스
```
cs/
  inbox/{msgId}     : source, bookingId, guest, lang, receivedAt, raw, parsed
  drafts/{msgId}    : reply, replyKo, category, confidence, status, editedReply
  sent/{msgId}      : 원문+최종답변+승인자+시각
  corpus/{id}       : 상황요약, 최종답변, lang, origin(구축/수정쌍/승인)
  tasks/{id}        : title, due, bookingId, source, status
  suggestions/{bid} : checkinTime?, specialRequests[], status
  guestScore/{bid}  : sentiment, msgCount, lastAt
  reviewQueue/{bid} : due, status
  config/           : autoSend(false), nightWindow(00-08), notifyChannel
```

## 8. UI (승인 웹앱)
- 현행 목업: cs_mockup_v0.3.html — PWR 아이보리 라이트 + 전광판만 다크(시그니처), Pretendard + Space Grotesk, 탭 5개(대기/업무/제안/후기/보냄), 자동응대 스위치 + 야간창 표기, 무수정 승인율 게이지.
- v0.4(컬러 배경 재도전)는 **M1과 병렬 진행** — 본질(파이프라인) 우선, 디자인이 M1을 막지 않는다.

## 9. 로드맵
- **M1 (수신 증명)**: 레포/GAS 생성 + Gmail 라벨·필터 + 1분 폴링 + 부킹 파서 + cs/inbox 적재.
  **M1 검증 항목**: ①부킹 알림메일 1건 파싱 정확도 ②회신 도달 1회 확인 ③**알림메일에 호스트(익스트라넷) 답변이 포함되는지** — 되면 "이미 처리" 자동 마킹 구현 가능.
- **M2 (생성·승인)**: 코퍼스 구축(CS-DB 임포트 + Gmail 마이닝) + Claude API 초안 + 승인 웹앱 + 푸시 + 승인 발송.
- **M3 (확장)**: 아고다·익스피디아 파서/발송 + suggestions + 업무 탭 + HK 버튼·뱃지.
- **M4 (자동화)**: autoSend 토글(야간창) + 후기요청 + 무수정 승인율 지표.

## 10. 미결정 잔여
